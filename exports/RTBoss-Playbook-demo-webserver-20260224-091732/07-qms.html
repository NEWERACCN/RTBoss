Voice & Tone

Precise, Not Jargon-Heavy

Say exactly what you mean. Use RTBoss terminology consistently (see Dictionary). Never use vague consulting language like "leverage synergies" — say "Finance OS and Sales OS share the same customer entity, so both Genies see the same client history in real time."

Outcome-First

Lead with the result, then explain the mechanism. Not "we built a Power Automate flow that triggers on invoice receipt" — instead "invoice processing now takes 4 minutes instead of 2 days — here's how."

Confident, Not Arrogant

We know what we're doing and we state it plainly. No hedge words like "we believe" or "hopefully." No puffery like "world-class" or "cutting-edge." State the architecture. State the outcome. Let the work speak.

Technical Depth When Needed

Adjust depth to audience. Board materials: outcomes and commercial model. Client materials: process and results. Engineering docs: full technical spec. Never dumb down — calibrate precision to context.

??
Mandatory on every output: Every document, Genie output, report, and data export must carry a classification label. This is an ISO 27001 Annex A control (A.5.12). No document leaves RTBoss without a classification. Agents that produce outputs must include classification in their output schema.

CONFIDENTIAL ?? CONFIDENTIAL Business sensitive. Authorized parties only. Not for general circulation. Encrypted in transit. RBAC enforced. Not forwarded externally without approval. Watermarked on print. Client SOWs, financial reports, agent system prompts, API credentials, HR performance data

Default rule: When in doubt, classify UP not down. An unlabelled document is treated as CONFIDENTIAL until reviewed. Genie outputs inherit the classification of their highest-sensitivity input — if a Genie processes RESTRICTED patient data, its output is RESTRICTED even if it's just a count or summary.

Advisors

Advisors are AI domain experts — deeply configured agents that provide strategic-level guidance within their specialty. Unlike Genies (which execute tasks), Advisors synthesise, interpret, and advise. Every client deployment activates the relevant Advisors for their industry and scale.

?
Advisor vs Genie distinction: Genies process and execute (extract this invoice, score this lead, generate this report). Advisors reason and advise (given our current cash position and Q3 pipeline, here is the capital deployment recommendation). Advisors are downstream of Genies — they consume processed facts, not raw inputs.

Reusable Knowledge Assets

Skills Catalogue

Skills are the bounded capability modules that Genies invoke to do their work. Where a Genie has a role, a mandate, and a persona, a Skill has only a single input and a single output. Skills are the reusable building blocks — write once, invoke everywhere. Every Skill in this catalogue is available to all Genies across all clients unless marked otherwise.

?
Skill vs Genie distinction: A Genie reasons, decides, and acts — it has a system prompt, a persona, and a mandate. A Skill just executes — pdf_extract(input_doc) ? structured_json. No persona. No mandate. No decision-making. Genies chain Skills together to complete their work. Skills can be called by multiple Genies — that's the point.
Skill Design Rules
RuleRequirementWhy
Single ResponsibilityEach Skill does exactly one thing. No compound Skills (e.g. "extract and validate" = two Skills).Skills must be reusable. Compound Skills can't be reused independently.
Typed InterfaceEvery Skill declares its input type and output type in the spec. Type mismatches raise a compile-time error, not a runtime failure.Genie build guides can be written without knowing which Skill version is installed.
StatelessSkills do not retain state between invocations. They receive input, produce output, return. No side effects.Stateless Skills are safe to retry, parallelise, and cache.
Idempotent Where PossibleCalling a Skill twice with the same input should produce the same output. Write Skills (bc_write, d365_write) are exceptions — they must include a deduplication key.Required for safe retry logic in Power Automate flows.
Risk-RatedEvery Skill is rated LOW / MED / HIGH risk. HIGH-risk Skills (those that write or delete data) require approval_workflow (SKL-010) to have run successfully in the same session before invocation.Prevents accidental data writes. Enforces human-in-the-loop for destructive operations.

Document & Content Skills · SKL-001 to SKL-009

SKL-001 pdf_extract

Extracts structured data from PDF documents using Azure AI Document Intelligence. Returns JSON with field names, values, confidence scores, and bounding box coordinates. Handles standard invoices, forms, and contracts. Custom extraction models supported.

  • Input: PDF binary or URL
  • Output: Structured JSON with confidence scores
  • Risk: LOW · No human approval required
  • Service: Azure AI Document Intelligence

SKL-002 excel_parse

Parses Excel workbooks (.xlsx, .xls, .csv) into structured JSON. Handles multi-sheet workbooks, named ranges, merged cells, and formula-derived values. Returns per-sheet array of row objects with header-keyed fields.

  • Input: Excel file binary or SharePoint URL
  • Output: JSON array of row objects per sheet
  • Risk: LOW · Used in Finance OS, Ops OS
  • Clients: All

SKL-007 document_summarise

Summarises a document to a specified persona and length. Output is shaped to persona_id — a summary for PRS-001 (MD) is a 3-bullet executive overview; for PRS-006 (Tech Lead) it's a full structured summary with technical detail preserved.

  • Input: Document text + persona_id + max_length
  • Output: Summary string shaped to persona
  • Risk: LOW · Read-only operation
  • Clients: All

SKL-008 table_extract

Extracts structured tables from documents, emails, or web pages. Returns array of row objects. Handles inconsistent column headers, merged cells, and footnote rows. Used primarily in Finance OS for statement parsing and Services OS for schedule extraction.

  • Input: Document text, HTML, or PDF URL
  • Output: Array of table row objects
  • Risk: LOW · Finance OS, Services OS
  • Clients: All

SKL-009 image_ocr

Optical character recognition for image files — PNG, JPG, TIFF, HEIC. Uses Azure Cognitive Services Computer Vision. Returns full text with line-level confidence. Used for handwritten form processing (HR OS) and invoice image capture (Finance OS).

  • Input: Image binary or URL
  • Output: Extracted text with confidence scores
  • Risk: LOW · Finance OS, HR OS
  • Service: Azure Cognitive Services

Performs a real-time web search and returns structured results with titles, snippets, URLs, and publication dates. Rate-limited per client per day to prevent runaway usage. SKL-010 approval_workflow

The gate Skill. Sends an approval request to the specified approver(s), waits for response, and returns approved/rejected with timestamp and approver identity. Must be invoked before any HIGH-risk write Skill. Approval tokens expire after 1 hour — a new approval must be obtained if the window passes. Never bypassed, never mocked in production.

  • Input: approver_persona_id[], request_description, timeout_hours
  • Output: {status: approved/rejected, approver_id, timestamp, token}
  • Risk: MED · Required before all HIGH-risk Skills
  • Token expiry: 1 hour from approval

Read-only connector to Microsoft Business Central. Returns financial records — invoices, purchase orders, GL entries, customer records. BC connector with read-only service principal scope.   • Risk: LOW · Read-only · Finance OS

QMS Toolbox Skills · SKL-018, SKL-019
??
QMS toolbox — highest risk class. These Skills write governance records — NCRs and controlled documents. They are never invoked autonomously. Every invocation requires: (1) SKL-010 approval_workflow completed, (2) human confirmation of the content, (3) full payload logged to neit_auditrecord. Misuse of these Skills constitutes a QMS compliance breach and raises an automatic NCR.

Requires human confirmation — never raised by a Genie without explicit human trigger.

  • Input: nc_source, description, severity, iso_clause, immediate_action, owner_id
  • Output: NCR ID (NCR-YYYY-nnn) + Dataverse record URL
  • Risk: HIGH · Requires human confirmation · Never autonomous
  • ISO: 9001 Cl.10.2 · 27001 Cl.10.1

F
Fact ENT-003 Data Model
The atomic unit of business intelligence in RTBoss. Every business event — a transaction, a decision, an enrichment — is represented as a Fact in Dataverse. Facts are immutable: they are never edited, only enriched (a new Fact is created referencing the original) or superseded (a correction Fact is linked to the original). This immutability is what makes the Digital Twin a reliable audit trail.
Example: A prescription arrives at EMS. Fact 1: "prescription received, source=fax, confidence=null." Fact 2: "prescription extracted by PDFExtractorAgent, confidence=0.94, patient_id=P-2891." Fact 3: "prescription validated by PharmacyCheckAgent, all fields confirmed."

G
Genie AI Agent
A discrete specialist agent that executes a single, clearly bounded task within the rules of its parent Mini-OS System. Genies are the workhorses of RTBoss — they process, transform, extract, validate, generate, or route. Every Genie has exactly one job. Genies never call other Genies directly — all multi-Genie coordination happens through the Master Orchestrator or a Stream.
Example: PDFExtractorAgent (AGT-002) has one job: extract structured data from incoming prescription PDFs and write a Fact to Dataverse. It doesn't validate the prescription (that's PharmacyCheckAgent). It doesn't trigger the dispensing workflow (that's the Master Orchestrator).

The top-level coordinating agent in every RTBoss deployment. Example: A prescription arrives. The Master Orchestrator receives the trigger ? routes to PDFExtractorAgent ? receives extracted data ? routes to PharmacyCheckAgent ? receives validation ? routes to InventoryAgent ? routes to BillingAgent. The Master knows the state of every step.

Certification & Audit Schedule

External Certification Audit

Annual surveillance audit by accredited certification body. Covers both ISO 27001 and ISO 9001. Preparation involves: current risk register review, NCR closure verification, internal audit completion, management review conducted, SoA current.

  • Stage 1: Document review (remote)
  • Stage 2: On-site/remote evidence assessment
  • Findings: minor/major NCs or observations
  • Output: Continued certification or corrective action required

Internal Audit Programme

Annual full-system internal audit. Additional triggered audits after: major incidents, significant system changes, new client onboarding, and before external certification. Auditor must be independent of area being audited.

  • Q1: Infrastructure and access controls (27001 A.8)
  • Q2: Delivery processes and customer satisfaction (9001 Cl.8–9)
  • Q3: Document control and records (both standards Cl.7.5)
  • Q4: Risk register and management review (both Cl.6–9)

Management Review

Quarterly meeting with MD. Reviews: internal audit results, NCR status, risk register, performance metrics, changes to context, and opportunities for improvement. Output logged and actioned.

  • Input: Audit findings, NCR log, risk register
  • Input: Section 06 performance metrics
  • Input: Customer satisfaction (NPS + outcomes)
  • Output: Decisions and improvement actions

Mandatory Pre-Deployment Checks

Before any new Genie or client deployment goes live, these ISO checks are mandatory gate criteria — not optional.

  • ISO control block complete and reviewed
  • Risk assessment conducted (ENT-005 record created)
  • Data classification assigned and documented
  • Test suite passed (all phases)
  • If personal data: Privacy Act assessment complete
  • Client DPA executed (if applicable)
